## 2. 前端服务器如何知道从哪个文件启动？

这是 Vue + Vite 的**约定配置**（Convention over Configuration）：

### 启动流程

```
npm run dev
    ↓
执行 vite 命令
    ↓
读取 vite.config.js（配置文件）
    ↓
默认查找项目根目录的 index.html
    ↓
index.html 中引用 <script src="/src/main.js">
    ↓
main.js 导入 App.vue 并挂载到 #app
    ↓
应用启动成功
```

### 关键文件

**`@g:\llm_swift\deepresearch-UI\index.html`** - 入口 HTML
```html
<div id="app"></div>
<script type="module" src="/src/main.js"></script>
```

**`@g:\llm_swift\deepresearch-UI\src\main.js`** - JavaScript 入口
```javascript
import { createApp } from 'vue'
import App from './App.vue'
createApp(App).mount('#app')  // 挂载到 <div id="app">
```

**`@g:\llm_swift\deepresearch-UI\src\App.vue`** - 主组件

所以启动顺序是：**index.html → main.js → App.vue → 其他组件**

## 3. 构建生产版本是做什么的？

### 开发环境 vs 生产环境

| 特性 | 开发环境 (`npm run dev`) | 生产环境 (`npm run build`) |
|------|-------------------------|---------------------------|
| **用途** | 本地开发调试 | 部署到服务器给用户使用 |
| **文件** | .vue 源代码 | 编译后的 .html/.js/.css |
| **大小** | 很大（包含开发工具） | 很小（压缩优化） |
| **速度** | 快速热更新 | 加载速度优化 |
| **端口** | localhost:5173 | 部署到你的域名 |

### `npm run build` 做什么？

1. **编译** - 把 `.vue` 文件转换成浏览器能理解的 HTML/CSS/JS
2. **压缩** - 删除空格、注释，缩短变量名
3. **打包** - 合并文件，减少 HTTP 请求
4. **优化** - Tree-shaking（删除未使用的代码）

### 输出结果

```
dist/
├── index.html          # 入口文件
├── assets/
│   ├── index-abc123.js   # 压缩后的 JS（带哈希值）
│   └── index-def456.css  # 压缩后的 CSS
```

这个 `dist/` 目录就是你要上传到服务器的**最终产品**。
